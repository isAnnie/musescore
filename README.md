# 前端乐谱编辑模块介绍

本模块是项目中的核心乐谱编辑前端，基于 Vue 3、TypeScript、Pinia、VexFlow 和 Tone.js 实现，提供从创建乐谱、导入文件、可视化编辑到播放试听与导出的完整编辑体验。

## 模块入口

乐谱编辑模块主要入口位于：

- `src/pages/EditorPage.vue`：编辑器页面入口，负责新建、导入、试听曲目、保存提示、导出等页面级流程。
- `src/components/editor/FullScoreEditor.vue`：完整乐谱编辑器容器，组织工具栏、画布、属性面板和播放控制。
- `src/components/editor/ScoreCanvas.vue`：乐谱画布，负责音符、谱表和编辑区域的可视化呈现。
- `src/stores/scoreStore.ts`：乐谱状态管理，负责乐谱创建、音符增删改、撤销重做、导入导出和本地持久化。

访问路径：

```text
/editor
/editor/:id
```

## 主要功能

### 新建乐谱

用户进入 `/editor` 后可以创建空白乐谱。新乐谱默认包含标题、作曲者、速度、拍号、调号、音符列表和草稿状态等基础信息。

默认参数包括：

- 标题：未命名乐谱
- 速度：120 BPM
- 拍号：4/4
- 调号：C
- 可见性：private

### 导入乐谱

编辑器支持从本地文件导入乐谱：

- `.musicxml`
- `.xml`
- `.mxl`
- `.mid` / `.midi` 入口已保留

其中 MusicXML 和 MXL 会被解析为编辑器内部的 `Score` 和 `Note` 数据结构，并映射到画布坐标，导入后可直接编辑和试听。

### 试听示例

`EditorPage.vue` 会自动扫描 `mxl/` 目录下的 `.mxl` 示例文件，并在“试听”面板中展示。用户可以选择示例曲目导入编辑器，用于快速体验乐谱渲染和播放效果。

### 可视化编辑

编辑器围绕五线谱画布组织交互，支持：

- 添加音符
- 更新音符属性
- 删除音符
- 调整音高、时值、位置等信息
- 维护高音谱号和低音谱号信息
- 处理休止符、附点、连音、升降号、踏板、装饰音等 MusicXML 信息

音符数据统一存储在当前乐谱的 `notes` 数组中，并通过 Pinia store 管理。

### 撤销与重做

`scoreStore` 内置历史记录能力：

- 每次编辑会保存当前乐谱快照
- 最多保留 50 条历史记录
- 支持撤销 `undo`
- 支持重做 `redo`

历史记录使用深拷贝保存，避免响应式对象继续变化污染旧状态。

### 本地持久化

编辑器会将乐谱状态写入 `localStorage`：

```text
musescore:score-store
```

持久化内容包括：

- 乐谱列表
- 当前乐谱 ID
- 最近编辑记录

当浏览器存储空间不足时，会降级保存最近乐谱，并限制大体量音符数据，避免写入失败影响编辑流程。

### 自动保存提示

编辑器页面包含自动保存开关和未保存状态判断。当前代码中服务端保存调用部分处于预留状态，保存和发布流程已保留接口位置，后续可接入真实后端保存逻辑。

### 播放与音频

模块使用 Tone.js 作为音频播放基础，编辑器内包含播放控制组件，用于试听当前乐谱。

### 导出能力

当前支持：

- MusicXML 导出
- MIDI 导出入口
- PDF 导出入口

其中 MusicXML 已有基础导出结构，MIDI 和 PDF 当前仍为占位或开发中。

## 数据结构

乐谱核心类型位于：

```text
src/types/score.ts
```

核心数据包括：

- `Score`：乐谱信息，包括标题、作曲者、速度、拍号、调号、音符、小节、创建时间、更新时间等。
- `Note`：音符信息，包括音高、时值、位置、谱号、声部、升降号、附点、连音、踏板和装饰音等。

编辑器通过统一的数据结构连接导入解析、画布渲染、属性编辑、播放和导出流程。

## 技术实现要点

- 使用 Vue 3 组合式 API 构建编辑器页面和组件。
- 使用 Pinia 管理当前乐谱、乐谱列表、历史记录和最近编辑状态。
- 使用 VexFlow 渲染五线谱和音符图形。
- 使用 Tone.js 提供浏览器端音频播放能力。
- 使用 JSZip 解压 `.mxl` 文件并读取其中的 MusicXML 内容。
- 使用 `localStorage` 做前端草稿和最近编辑的轻量持久化。

## 当前限制

- MIDI 导入目前是入口保留，尚未接入完整解析能力。
- MIDI 导出和 PDF 导出仍处于开发或占位状态。
